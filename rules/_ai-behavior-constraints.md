# AI 行為約束

> **自動載入**: 此檔案位於 `~/.claude/rules/`，會自動載入到所有會話

## 🔴 強烈警告：AI 禁止行為

> **這些行為會破壞 CLAUDE.md 的實用價值，絕對禁止**

### ❌ 絕對禁止的行為

| 行為 | 原因 | 後果 |
|------|------|------|
| **加入統計資訊** | 行數/字數是快照，每次修改都過時 | AI 被誤導，以為程式碼很小 |
| **加入版本號** | AI 不關心 v1.0 → v2.0 | 浪費 token，無實質資訊 |
| **模組描述標版號** | 如 `ClassName（v3: ...）`必然過時 | 功能性描述始終準確 |
| **加入更新日期** | AI 只需要「當前」規則 | 干擾核心內容 |
| **加入 Changelog** | 歷史變更應放 CHANGELOG.md | 文檔膨脹 |
| **作者資訊** | 除非有特殊意義 | 無聊且干擾 |
| **寫入可推導內容** | API 簽名、參數表、欄位列表可從程式碼直接推導 | 浪費 token，降低 signal/noise ratio |
| **完整程式碼範例 (>5 行)** | 應精簡為一句話描述 + 源碼引用 | 文檔膨脝，維護成本高 |

### ✅ 正確的 AI 行為

| 行為 | 原因 | 價值 |
|------|------|------|
| **專注核心原則** | AI 需要知道「為什麼」 | 指導決策 |
| **暴露結構關係** | AI 需要了解檔案依賴 | 理解專案 |
| **說明當前狀態** | 現有的規則和約束 | 執行依據 |

## 🔴 Bash 強烈警告：`python -c` 禁止寫註解

> **`python -c` 是 AI 自用驗證，不需人類可讀註解。** 加了 `#` 註解會觸發權限確認卡關 — Claude CLI 無法判斷 newline + `#` 的跨行註解是否被注入惡意內容，故每次都需人工確認。要驗證想法就寫乾淨單行，或落成 `.py` 檔。完整 Bash 硬限制見 `bash-hard-rules.md`。

## 第一性原理分析

### 為什麼統計資訊對 AI 無價值？

#### 時點性問題
- **行數是程式碼變動的快照**
- 每次修改都會過時
- AI 讀取時行數已經不同

#### 認知負擔
- AI 看到 `行數: 100` 可能以為是小檔案
- 實際可能有 10000 行在檔案其他地方
- **造成錯誤判斷**

#### 維護成本
- 每次修改都要更新統計
- 實用價值接近零

#### 對 AI 真正有價值的資訊

✅ **檔案存在性** - 檔案在哪裡
✅ **職責劃分** - 每個模組做什麼
✅ **設計理由** - 為什麼這樣做（從程式碼猜不到的）
✅ **關鍵約束** - 不可妥協的規則
✅ **引用路徑** - 相關檔案位置

### 為什麼模組描述不標版號？

#### 版號是過時實踐

| 問題 | 說明 |
|------|------|
| **Git 已解決版本問題** | `git log` 是權威版本記錄，文檔版號必然不同步 |
| **LLM 不消費版號** | LLM 需要功能性描述（做什麼、怎麼用），版號不提供可操作資訊 |
| **維護成本 > 價值** | 每次修改都需手動更新，忘了就成為誤導資訊 |
| **各模組版號各自演化** | 多模組各自標 v3/v6/v7，無法對齊，最終全過時 |

#### 正確做法：功能性描述取代版號

```
# ❌ 禁止：版號標記（必然過時）
├── setup_classifier.py  # SetupClassifier（v7: pure AND/OR evaluator）

# ✅ 正確：功能性描述（始終準確）
├── setup_classifier.py  # SetupClassifier（pure AND/OR evaluator, multi-setup, YAML-driven）
```

#### 唯一例外

API 中的實際方法名或常數名包含版號時（如 `full_pipeline_v2()`），如實引用。

### 結論

**移除統計表格是正確決策**
- 減少 token 消耗
- 避免 AI 被誤導
- 專注於當前有效的規則

## 執行約束

### 寫作 CLAUDE.md 時必須遵守

1. **不加入任何元資訊區塊**
2. **不統計行數或字數**
3. **不標註版本或日期**
4. **專注於「當前有效」的規則**

### 修改 CLAUDE.md 時必須遵守

1. **移除發現的元資訊**，不保留
2. **不添加新的元資訊**，即使其他檔案有
3. **使用 `/claude:clean` 驗證**
