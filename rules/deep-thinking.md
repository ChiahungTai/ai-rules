# 深層思考框架

> **自動載入**: 此檔案位於 `~/.claude/rules/`，會自動載入到所有會話

---

## 核心原則

**所有決策必須經過兩層思考：第一性原理（向下分解，找到本質）+ 第二層思考（向前追蹤，預見後果）。**

只做第一性原理分析，會產生「本質正確但後果災難」的決策（如 Tesla 生產地獄：製程設計從物理原理出發完全正確，但未追蹤過度自動化在真實工廠環境的連鎖後果）。

---

## 思維框架

```
第一性原理（向下分解）                  第二層思考（向前追蹤）

1. 追溯基本問題 → 本質需求是什麼？不可妥協的約束？
2. 拆解到基本元素 → 基本組件與本質職責
3. 從基本原理重構 → 最佳方案？本質權衡？
4. 驗證假設 → 查證事實，根據證據調整
                                  ↓
5. 追蹤連鎖後果 → 「接下來會怎樣？」至少追蹤兩層（強制）
6. 評估完整影響 → 負面後果可接受嗎？需要重新分解嗎？
                                  │
                       後果不可接受 → 回到步驟 1 ─┘
```

### 第二層思考的強制性

**步驟 5-6 是強制的**，所有決策都必須追蹤至少一層後果。LLM 再判斷是否需要更深入分析（第三層、第四層）。

**判斷更深分析的信號**：
- 決策影響多個模組/系統
- 決策修改公開 API 或跨專案方法論
- 第一層後果中出現「可能」、「或許」等不確定性
- 歷史上同類決策曾經出錯

## 決策分級

| 決策類型 | 思考深度 | 判斷標準 |
|---------|---------|---------|
| **單向門**（不可逆） | 第一性原理 + 多層後果追蹤 | 錯了無法逆轉：DB migration、公開 API 合約、跨專案方法論、生產架構 |
| **雙向門**（可逆） | 第一性原理 + 第二層後果追蹤 | 錯了可以回退：內部重構、開發工具、配置調整 |

## 關鍵問題

### 第一性原理層（本質是什麼？）

1. **本質需求** — 必須做到什麼？不可妥協的約束？
2. **為什麼需要** — 解決什麼本質問題？沒有它會怎樣？
3. **權衡** — 每個選擇用什麼換取什麼？
4. **驗證** — 什麼事實會推翻這個設計？

### 第二層思考（接下來會怎樣？）

5. **第一層後果** — 這個決策的直接影響？
6. **第二層後果** — 第一層影響又會導致什麼？
7. **如果我是錯的** — 最壞情況？可以逆轉嗎？逆轉成本？

## 領域應用

### 軟體工程

架構決策前必須回答：

- **本質需求**：這個系統/模組必須做到什麼？
- **為什麼需要**：沒有它會發生什麼災難性後果？
- **查證**：基於實際程式碼驗證，不依賴 LLM 訓練資料

**程式碼查證**：
- 專案程式碼：直接讀取源碼
- 套件源碼：從 `.venv/lib/python*/site-packages/` 讀取，確認實際行為
- 引用具體路徑（如 `.venv/lib/.../main.py:122`），不憑記憶

**連鎖後果追蹤**（軟體實例）：
- 修改 A 模組 → 依賴 A 的 B/C/D 模組是否受影響？→ B 的下游消費者是否受影響？
- 新增抽象層 → 減少了什麼複雜度？→ 引入了什麼新的認知負擔？
- 改名/重構 → 所有引用處是否更新？→ 文檔是否同步？→ 下游專案是否斷裂？

### 股票分析

**分析必備視角（由下而上，每一層都必須經過）**：

**Layer 1（必備）：經典技術分析**

所有股票分析必須先從經典技術分析理論出發，作為判斷的基礎框架：

- **Dow Theory**：主趨勢/次趨勢/日間波動三層結構、量價確認、趨勢持續直到明確反轉信號
- **Elliott Wave**：推動波（5 波）/修正波（3 波）結構、Fibonacci 比率（0.382, 0.5, 0.618, 1.618）
- **Wyckoff**：累積/markup/distribution/markdown 四階段、供需分析、Composite Operator 行為
- **Darvas Box**：箱型整理突破、量價配合確認、向上突破買進、向下跌破停損
- **Bollinger Bands**：均值回歸vs趨勢延續判斷、頻寬收縮（squeeze）預示突破、價格觸帶非訊號需搭配其他指標
- **Support/Resistance**：關鍵價位（前高前低、整數關卡、缺口）、支撐阻力互換
- **Volume Analysis**：量先價行、量價背離、OBV、累積/派發
- **Trend Analysis**：高點低點關係、趨勢線與通道、趨勢的三種規模

**為什麼經典 TA 必須先於量化指標**：經典理論提供的是**市場結構的解釋框架**（為什麼會這樣走），量化指標提供的是**數值化描述**（目前數值是多少）。先有解釋框架，數值才有意義。RSI=75 在強趨勢中（Dow Theory 主趨勢向上）可以長期維持，不代表超買。

**Layer 2（疊加）：量化/系統觀點**

經典 TA 框架建立後，疊加系統計算的指標和結構：
- `mosaic_alpha/indicators/` — 40+ 技術指標（RSI, MACD, ATR 等）
- `mosaic_alpha/structure/` — WaveNode, SwingLeg, TrajectoryTree
- `mosaic_alpha/features/` — Spec-driven 特徵工程

**Layer 3（整合）：第二層思考**

用第二層思考整合多維度分析。**核心動作：列出各技術分析理論之間的共識與歧異，讓用戶判斷**：

- **共識點**：哪些理論對當前走勢有相同判斷？（共識越強，可靠性越高，但也意味著已被定價）
- **歧異點**：哪些理論對當前走勢有不同判斷？歧異在哪裡？（歧異處是風險，也是機會）
- 「這個價位反映了什麼預期？什麼事實會打破這個預期？」
- 「如果這個判斷是錯的，最大損失是什麼？我接受嗎？」
- 「風險的悖論：所有人都認為安全時最危險，所有人都認為危險時可能最安全」

**輸出要求**：股票分析結論必須包含「各理論共識/歧異對照」段落，格式：

```markdown
### 理論共識與歧異
| 理論 | 判斷 | 信心 | 依據 |
|------|------|------|------|
| Dow Theory | 偏多 | 高 | 主趨勢向上，量價配合 |
| Elliott Wave | 偏空 | 中 | 可能處於第 5 波末端 |
| Bollinger | 中性 | 低 | 頻寬收縮，方向未明 |
| Darvas Box | 偏多 | 中 | 箱頂突破確認 |
| **共識** | 偏多（3:1） | — | 主要理論偏多，但 Elliott Wave 歧異需注意 |
```

## 觸發情境

### 必須觸發深層思考

1. **架構變更**：新增、移除、或重構系統組件
2. **技術選型**：選擇技術方案、工具、或框架
3. **設計決策**：影響系統結構的重大決定
4. **性能優化**：涉及架構層級的優化方案
5. **重構建議**：AI 主動發現問題並提出重構
6. **股票分析**：任何涉及個股或市場走勢的判斷
7. **AI 建議評估**：評估其他 AI 的設計建議或審查意見
