# 股票分析框架

> **載入時機**：分析個股或市場走勢時按需載入。通用深層思考框架（第一性原理 + 第二層思考）見 deep-thinking rule，本 skill 是其股票領域的具體應用。

## 分析必備視角（由下而上，每一層都必須經過）

### Layer 1（必備）：經典技術分析

所有股票分析必須先從經典技術分析理論出發，作為判斷的基礎框架：

- **Dow Theory**：主趨勢/次趨勢/日間波動三層結構、量價確認、趨勢持續直到明確反轉信號
- **Elliott Wave**：推動波（5 波）/修正波（3 波）結構、Fibonacci 比率（0.382, 0.5, 0.618, 1.618）
- **Wyckoff**：累積/markup/distribution/markdown 四階段、供需分析、Composite Operator 行為
- **Darvas Box**：箱型整理突破、量價配合確認、向上突破買進、向下跌破停損
- **Bollinger Bands**：均值回歸vs趨勢延續判斷、頻寬收縮（squeeze）預示突破、價格觸帶非訊號需搭配其他指標
- **Support/Resistance**：關鍵價位（前高前低、整數關卡、缺口）、支撐阻力互換
- **Volume Analysis**：量先價行、量價背離、OBV、累積/派發
- **Trend Analysis**：高點低點關係、趨勢線與通道、趨勢的三種規模

**為什麼經典 TA 必須先於量化指標**：經典理論提供的是**市場結構的解釋框架**（為什麼會這樣走），量化指標提供的是**數值化描述**（目前數值是多少）。先有解釋框架，數值才有意義。RSI=75 在強趨勢中（Dow Theory 主趨勢向上）可以長期維持，不代表超買。

### Layer 2（疊加）：量化/系統觀點

經典 TA 框架建立後，疊加系統計算的指標和結構（以量化交易專案為例，跨專案時替換為當前專案的對應模組）：
- `mosaic_alpha/indicators/` — 40+ 技術指標（RSI, MACD, ATR 等）
- `mosaic_alpha/structure/` — WaveNode, SwingLeg, TrajectoryTree
- `mosaic_alpha/features/` — Spec-driven 特徵工程

### Layer 3（整合）：第二層思考

用第二層思考整合多維度分析。**核心動作：列出各技術分析理論之間的共識與歧異，讓用戶判斷**：

- **共識點**：哪些理論對當前走勢有相同判斷？（共識越強，可靠性越高，但也意味著已被定價）
- **歧異點**：哪些理論對當前走勢有不同判斷？歧異在哪裡？（歧異處是風險，也是機會）
- 「這個價位反映了什麼預期？什麼事實會打破這個預期？」
- 「如果這個判斷是錯的，最大損失是什麼？我接受嗎？」
- 「風險的悖論：所有人都認為安全時最危險，所有人都認為危險時可能最安全」

## 輸出要求

股票分析結論必須包含「各理論共識/歧異對照」段落，格式：

```markdown
### 理論共識與歧異
| 理論 | 判斷 | 信心 | 依據 |
|------|------|------|------|
| Dow Theory | 偏多 | 高 | 主趨勢向上，量價配合 |
| Elliott Wave | 偏空 | 中 | 可能處於第 5 波末端 |
| Bollinger | 中性 | 低 | 頻寬收縮，方向未明 |
| Darvas Box | 偏多 | 中 | 箱頂突破確認 |
| **共識** | 偏多（3:1） | — | 主要理論偏多，但 Elliott Wave 歧異需注意 |
```
