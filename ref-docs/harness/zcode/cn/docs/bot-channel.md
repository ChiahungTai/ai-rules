Bot Channel
Bot Channel 用来把外部聊天工具接入 ZCode。配置完成后，你可以从微信或飞书会话里打开当前工作区，查看任务进度，并继续给 Agent 发送指令。
和 Remote Control 的区别
Remote Control 更适合临时扫码接入；Bot Channel 更适合把入口放进常用聊天工具里，便于更长时间回访同一个工作区。
Remote Control：手机扫码，快速打开当前工作区。
Bot Channel：从微信或飞书会话进入工作区。
新建 Bot Channel
打开 移动端远程控制 弹窗，右侧就是 Bot Channel 入口：点击对应聊天工具开始接入，或进入底部 机器人管理 查看已有机器人。
目前可创建的渠道包括：
微信：扫码登录，自动绑定。
飞书：扫码创建应用，再通过消息绑定。
钉钉、Discord 和企业微信会在后续版本继续补齐。
微信配对流程
微信的接入最简单：在机器人页选择 微信，用手机微信扫码并确认连接，凭据会自动保存。绑定完成后，直接在微信里和这个 Bot 对话即可。
飞书配对流程
飞书分两步。第一步创建应用：在 ZCode 里选择 飞书 并扫码，手机上会打开飞书开放平台，确认智能体应用的名称和头像后点击 立即创建，等待配置完成。
第二步绑定会话：ZCode 会生成一段限时有效的 绑定命令（形如 /bind C67356），在飞书里向这个智能体发送该命令，机器人回复绑定成功并列出全部可用命令。
绑定成功后，飞书 Bot 会返回可用命令。常用命令包括查看状态、新建任务、切换项目、切换模型、切换运行模式和调整回复详细度。后续你可以直接在飞书里向 ZCode Agent 发消息，适合在手机上持续跟进长任务。
管理机器人
创建后，机器人会出现在左侧列表中。这里可以启用或停用机器人、查看连通状态、设置回复颗粒度，并限制它可以访问的 workspace。
常用配置包括：
关联机器人：扫码或按界面提示完成凭据绑定，已连通的机器人可随时 解绑。
机器人回复颗粒度：控制机器人回复的详细程度。
工作区访问范围：限制机器人可打开的工作区。
删除机器人：移除不再使用的渠道。
从聊天工具进入工作区
Bot Channel 配置完成后，就可以在聊天工具中向机器人发送消息。机器人会把请求转给 ZCode Agent，并返回任务进度或工作区入口。
这种方式适合在手机上长期跟进任务：你可以在微信或飞书里继续发消息，必要时再打开工作区查看完整会话。
