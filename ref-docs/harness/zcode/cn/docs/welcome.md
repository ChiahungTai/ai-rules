欢迎使用 ZCode for GLM-5.2
ZCode 是一个把 GLM-5.2 带入真实编程工作流的 Agentic Development Environment (ADE)，帮助你把 GLM-5.2 的长上下文、长程任务与 Agentic 编程能力，转化为稳定可用的桌面体验，覆盖复杂开发任务中的规划、编码、审查与迭代。
依托 GLM-5.2 稳定的 1M 上下文与长程任务能力，ZCode Agent 可以把目标、文件、终端结果、浏览器上下文、执行模式和 Git 状态保持在同一个任务里，让复杂开发任务从规划一路推进到实现与验证收口，而不丢失连续性。
ZCode 围绕 GLM-5.2 完成深度联调与专项优化，聚焦自研 ZCode Agent 能力。模型、工具和执行工作流结合更紧密，让复杂开发任务可以在同一个上下文里从规划一路推进到验证收口。
通过自然语言指令，你可以让 ZCode Agent 处理编码、调试、测试、项目预览和变更审查等工作；通过桌面端工作区、Remote 和 Bot Channel，也可以在长任务执行中持续查看进度并补充指令。
核心能力
ZCode 当前围绕自研 ZCode Agent 组织开发体验，并针对 GLM-5.2 系列模型做了深度适配，重点强化以下能力：
GLM-5.2 深度联调：围绕模型能力、工具调用和执行链路进行多轮打磨，让 Agent 更适合连续、多步骤的真实开发任务。
长任务执行：在同一个任务里完成需求理解、计划拆解、代码修改、验证和 Review，减少频繁切换上下文带来的中断。
稳定上下文保持：结合 GLM-5.2 的长上下文能力，持续读取文件、终端、浏览器、执行模式和 Git 状态，提升长程任务的持续推理能力。
自研 Agent 工作流：任务、权限、上下文、工具调用和提交流都围绕 ZCode Agent 组织，模型输出可以更自然地落到实际工程操作中。
多端持续跟进：桌面端、手机端 Remote 与飞书 / 微信 Bot 可以共同推进同一个工作区任务。
安全可控：关键命令、文件修改和高权限操作会在执行前进入确认流程。
GLM-5.2 与专属权益
GLM-5.2 已在 ZCode 中全量开放，支持真正可用的长上下文能力，适合需求跨度更长、文件更多、需要持续推理的开发任务。ZCode Agent 会在任务执行过程中持续结合工作区状态、工具结果和 Git 变更，让模型能力更稳定地转化为代码改动。
