编辑历史对话
编辑历史对话用于修正已经发送给 ZCode Agent 的消息。当你发现前一条指令写得不够准确、缺少文件路径、需要补充约束，或想调整任务方向时，可以直接编辑原消息，让 Agent 按新的上下文重新理解任务，而不用重新创建任务。
这对长任务尤其有用：你可以保留当前任务里的模型、工作区、分支和上下文，只修改关键指令，让后续执行围绕更新后的目标继续推进。
找到编辑入口
在对话历史记录中，将鼠标悬停在自己发送过的消息上。消息右侧会显示快捷操作区，其中铅笔图标就是 编辑 入口；旁边也会保留复制等常用操作。
入口特点
只出现在用户消息上：编辑入口用于修改你已经发送的指令，不会影响 Agent 的回复内容本身。
作用于最近一轮：当前可编辑的是最后一轮的用户消息，任务运行中或有排队消息时暂不可编辑。
适合快速纠偏：发现目标、路径或约束有误时，不需要重新开一个任务。
编辑并重新发送
点击编辑按钮后，原消息会展开为可编辑输入框。你可以直接改写文字，也可以继续使用 @ 引用文件、使用 / 插入命令。确认无误后点击发送，ZCode Agent 会按更新后的消息继续处理任务；如果不想保留修改，点击取消即可退出编辑状态。
操作逻辑
进入编辑：点击用户消息右侧的铅笔图标。
修改内容：在展开的输入框里调整原始指令，补充文件、路径、目标或限制条件。
提交或取消：点击发送让 Agent 基于新指令继续推进；点击取消则保留原消息不变。
使用场景
修正需求表达：发现前一条指令写错、遗漏目标或描述不够清楚。
补充关键上下文：追加文件路径、错误日志、截图说明、接口参数等信息。
调整任务方向：让 Agent 从“分析问题”改为“直接修复”，或从“实现功能”改为“先给方案”。
减少重复对话：避免重新解释整段背景，让任务在同一个上下文里继续前进。
